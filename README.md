## 🧩 How I Approach Each Problem

1. **Understand** — break down constraints and edge cases
2. **Brute force first** — get a working solution, understand the bottlenecks
3. **Optimize** — identify what's slow and why, then fix it deliberately
4. **Document** — write a `notes.md` with the full thought process

The goal isn't just an accepted solution. It's understanding _why_ one approach beats another.

---

## 🚀 Problem Solving Philosophy

- **Start with correctness** — get a working solution before optimizing
- **Understand the bottleneck** — identify _why_ it's slow before changing anything
- **Optimize deliberately** — every change should have a clear reason behind it
- **Document the journey** — the path from brute force to optimal is where the real learning happens
